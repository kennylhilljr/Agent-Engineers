WHAT WAS ADDED / CHANGED
--------------------------
File: /Users/bkh223/Documents/GitHub/agent-engineers/exceptions.py

1. AgentError.to_dict() - Enhanced to include __cause__ chain:
     - When __cause__ is set, a "cause" key is added to the dict.
     - If cause is an AgentError subclass, its own to_dict() is used recursively.
     - Otherwise, error_type and message are extracted from the plain exception.

2. AgentError.to_json() - New method:
     - Returns json.dumps(self.to_dict()) for direct JSON string output.

3. ConfigurationError(AgentError) - New class:
     - Default error_code: "CONFIG_ERROR"
     - Optional config_key attribute (included in to_dict when set)

4. TimeoutError(AgentError) - New class:
     - Default error_code: "TIMEOUT_ERROR"
     - Optional timeout_seconds attribute (float, included in to_dict when set)

5. RateLimitError(AgentError) - New class:
     - Default error_code: "RATE_LIMIT_ERROR"
     - Optional retry_after (float) and provider (str) attributes

6. AuthenticationError(AgentError) - New class:
     - Default error_code: "AUTH_ERROR"
     - Optional username attribute (included in to_dict when set)

7. is_retryable(exc) -> bool - New module-level helper:
     - Returns True for RateLimitError and TimeoutError instances (by type)
     - Returns True for AgentError instances whose error_code is in the
       known retryable set: BRIDGE_CONNECTION, BRIDGE_RATE_LIMIT, BRIDGE_TIMEOUT,
       TIMEOUT_ERROR, TIMEOUT_REQUEST, TIMEOUT_OPERATION, TIMEOUT_CONNECTION,
       RATE_LIMIT_ERROR, RATE_LIMIT_EXCEEDED, RATE_LIMIT_PROVIDER, RATE_LIMIT_INTERNAL
     - Returns False for all other exceptions

8. get_error_code(exc) -> Optional[str] - New module-level helper:
     - Returns exc.error_code if exc is an AgentError, else None.

9. error_handler() context manager - New @contextlib.contextmanager function:
     - Intercepts AgentError exceptions.
     - reraise=True (default): calls log_fn if provided, then re-raises.
     - reraise=False: suppresses the exception after calling log_fn.
     - Non-AgentError exceptions propagate unaffected in either mode.
     - Yields an _ErrorHandlerContext whose .exception attribute is set to the
       caught AgentError (or None if no exception occurred).
