OPTIMIZATIONS IMPLEMENTED
--------------------------
1. LRUCache (OrderedDict-backed, O(1) get/set/evict)
   - TTL-based expiry for automatic cache invalidation
   - Configurable maxsize (default 128)
   - invalidate() for manual key removal
   - clear() for full cache reset
   - stats property for observability

2. cache_result decorator
   - Wraps any function with automatic result caching
   - MD5-keyed on (args, sorted kwargs) for cache-safe hashing
   - TTL support for time-bounded caching
   - Preserves __name__ and __doc__ via functools.wraps
   - Exposes _cache attribute for inspection/testing

3. AsyncConnectionPool
   - asyncio.Queue-based pool for reusable connections
   - Configurable max_size and acquire timeout
   - Prevents repeated TCP/subprocess connection setup overhead

4. TokenCache
   - Stores API tokens with monotonic-clock expiry
   - Per-provider invalidation and global clear
   - Default 3600s (1 hour) TTL matching typical OAuth token lifetimes

5. Global singletons
   - get_api_cache(): LRUCache(maxsize=256, ttl=300) for API responses
   - get_token_cache(): TokenCache() for authentication tokens
