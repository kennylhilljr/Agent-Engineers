## Manual Setup Instructions

### Step 1: Create Team (if needed)
```
Linear UI → Teams → Create new team "Agent Dashboard"
```

### Step 2: Create Project
```
Linear UI → Projects → New Project
Name: Agent Dashboard
Team: Agent Dashboard
Description: Web-based dashboard for real-time agent monitoring, AI chat interface, and agent controls
```

### Step 3: Create META Issue
```
Title: [META] Project Progress Tracker
Type: Issue
Team: Agent Dashboard
Description: Session handoff and project coordination. Use to track progress, blockers, and session context.
Label: meta, coordination
Status: Backlog
```

### Step 4: Create Epic Issues (one per phase)
For each phase (1-6):
```
Title: Phase N: [Phase Name]
Type: Epic
Team: Agent Dashboard
Description: [From Phase definition above]
Status: Backlog
```

### Step 5: Create Feature/Task Issues
For each requirement:
```
Title: [REQ/QA-ID]: [Requirement Title]
Type: Feature or Task
Team: Agent Dashboard
Parent: [Corresponding Epic or none for QA]
Description: [Full description with test steps from template]
Labels: [phase-N or qa]
Status: Backlog

Special Note: For issues using reusable components, add to description:
"Reusable component available: a2ui-components/<component-name> — adapt instead of building from scratch"
```

### Step 6: Add Initial Comment to META Issue
```
Comment: "Session 1 initialized. Created 71 issues across 6 implementation phases plus QA improvements. See linked issues for detailed requirements."
```

## Integration with Development

### Deduplication Strategy
- After creating issues, verify no duplicates exist
- Maintain `.linear_project.json` state file
- When adding new issues, check existing titles (lowercased)

### Session Handoffs
- Use META issue for coordination
- Add comments summarizing session work
- Link related issues
- Mark blockers with "blocker" label

### Verification Workflow
After each phase completion:
1. Run acceptance tests
2. Update issue status to "In Progress" → "In Review" → "Done"
3. Comment with verification results
4. Update `.linear_project.json` state file
5. Link to related PRs/commits

## State File Format

The `.linear_project.json` file tracks project metadata:

```json
{
  "project_id": "PROJECT_ID",
  "project_name": "Agent Dashboard",
  "meta_issue_id": "ISSUE_ID",
  "meta_issue_key": "AI-1",
  "issues": [
    {
      "key": "AI-1",
      "title": "[META] Project Progress Tracker",
      "status": "Backlog"
    },
    ...
  ],
  "total_issues": 71,
  "done_issues": 0,
  "last_verification_status": null,
  "tickets_since_verification": 0
}
```
